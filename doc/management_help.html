1. Using the “MS Data Management” module

Once you have selected the MS Data Management module from the ProHits access page and have logged in, the following screen will appear.

At the top of the page (left), you will find the tabs “Home”, “Storage” and “Auto Search” that allow you to navigate between the storage and the search areas of the Data Management Module. On the left is a link to the Analyst module that can be accessed from every page of the Data Management module.


The “Storage” section allows you to monitor the transfer of data from each of the acquisition computers to the ProHits backup system, and to assist in preparing an activity report for the use of a mass spectrometry facility. It also allows you to search, browse and download backed up files, convert raw files to other formats, and manually upload raw data. Export of files through FTP, including generation of all files for ProteomeXchange submissions in MassIVE, is also available under the storage tab.

The “Auto Search” section allows you to perform and schedule database search tasks on specified files using user-defined search engines and parameters, to explore the results and to link search results to the experimental information in the Analyst module. Data Dependent Acquisition and Data Independent Acquisition workflows are both supported.


2. Storage

Navigating the Storage home


=> Select the “Storage” tab

ProHits manages the backup and storage of data files in an instrument-dependent manner. The left of this screen provides links to all available instruments in the mass spectrometry facility (alternatively the user can click on the instrument's image). This page also specifies the location of the database and data storage. The central part of the page details the status of each of the connections. A green arrow indicates a functional connection while a broken orange arrow denotes a broken connection. Note that there are additional links toward the bottom of the left column (Fetch Raw File, Raw file status, Backup log and Export Raw files); these will be briefly discussed at the end of this “Storage” section.



Selecting instruments and folders

The file backup organization mirrors the set-up on each acquisition computer (the computer linked to the MS instrument). Subfolders are allowed on the acquisition computer. Selecting the instrument will open a new page allowing you to browse files and projects. In the demo version, only one instrument (LTQ_demo) is available.


=> Select an instrument by clicking on a link on the left of the page


At the bottom of the page, the folders associated with this instrument are listed (along with unique identifier assigned by ProHits). Folders are associated to individual “Analyst” projects defined in the Admin Office (here, folders associated with project P84 are displayed). Note that adding a suffix to the folder name “_P84” enables the creation of Autolinks (as will be discussed later).


The Search Task column allows you to navigate to the search results page(s) associated with files in this folder. In the Options column, clicking will open the folder, while selecting will download the files associated to this folder on your local computer.

ProHits automatically backs up any new files from the acquisition computer at a time specified by the administrator.

The user may also initiate manual back up and by clicking [Backup File Now] (towards the top of the page). Please note that to prevent file corruption, ProHits has a timed delay for file transfer (delay specified by administrator – at LTRI we use 2.5 hours after the last modification).

To upload raw files from an instrument not connected to the ProHits backup system, select [Upload Raw Files] and navigate through options.

=> Select a folder to open

In addition to the files located on the acquisition computer, the selected folder will also contain any files that you have converted from an original raw file to other formats, and any file types stored on the acquisition computer (e.g. .doc, .xls, etc.). This folder will also contain any file manually uploaded on the previous page.


Data file conversion

ProHits can convert vendor-specific raw files to database-search-engine-preferred file types as part of the Auto Search pipeline. Alternatively, raw files can easily be converted here (this is enabled primarily through ProteoWizard).

To convert a raw file to mgf, mzXML, mzML or mz5 format, choose the desired files by clicking the boxes in the “Convert” column, select the desired format and hit “Convert”. The “Download” column allows you to save the data on your local computer. wiff/scan file types can be converted to .mgf in a similar manner.


Linking files to Analyst

As mentioned in the general introduction, raw files located in the Data Management module can be linked to baits in the Analyst module and the search results can then be transferred (parsed). Each file can be linked either automatically or manually to an entry in the Analyst module. This is done simply by clicking on a white “link” icon () in the “project Bait/sample” column. This opens up a new dialog box allowing you to select the desired Project, Bait, and Sample. Upon closing the box, a yellow chain link () will appear. Creating automatic links (shown as green “link” icons; ) requires interfacing with the Analyst module and using a standardized naming scheme for both the samples/raw files and folders/projects. This will be described in a separate section. Creation of New Baits and Samples will be described in the instructions for the Analyst module.


=>Select the “Fetch Raw file” link (bottom left)

This permits you to search for a specific file name, and restrict by date and instrument.



=> Select the “Raw file status” link (bottom left)

This permits you to generate an activity report for the instrument(s) in your mass spectrometry facility.



=> Select the “Export Raw Files” link (bottom left)

This permits you to download a set of files to your local computer, transfer them to other storage folders on ProHits or export them via FTP. This section also assists with the deposition of files for complete submissions to the ProteomeXchange server via the MassIVE mass spectrometry data repository.


This completes the overview of the “storage” part of the MS Data Management module of ProHits. In the next sections, we will navigate through the searching and parsing functions for the Data Dependent and Data Independent Acquisition files modes.


3. Auto Search for Data Dependent Acquisition

Navigating the Auto Search home

=> Select the “Auto Search” tab at the top of the page

This view displays all the search engines and other tools that have been linked to your ProHits database in the top portion of the page. As with the link to different mass spectrometers in the Storage area, successful links to search engines are indicated by green arrows and broken links by a broken orange arrow. The bottom part of the page allows you to access individual instruments for search purposes.



=> Select an instrument for searching

The entry page lists all of the search tasks that were performed for files collected on a given instrument. The tasks are named by individual scientists searching their data. A given task may be applied to several files not necessarily located within the same subfolder or in the same format. We will return to this list later after we have created a New Task.


New Task function for Data Dependent Acquisition

=> Select “Instrument New Task”

This opens up a new page; by default, ProHits will open the search pipeline for Data Dependent Acquisition. To instead search Data Independent Acquisition files, click the “Is DIA file” button (this will be reviewed in the next section).


Enter a Task Name of your choice. Select the search engine(s), database, modifications and parameter set to be utilized (these can be edited as shown below). Alternatively, parameter sets can be imported from other search tasks.

Parameter set: This provides, for each search engine, thegeneral parameters to be used. The parameters are instrument specific (Parameters must be specified on each instrument). The example here displays the Comet parameters on a TripleTOF instrument:

Search engines: Currently, ProHits can support Mascot, X!Tandem, Comet and MS-GF+ for searching Data Dependent Acquisition files or pseudo-MS/MS files derived from DIA-Umpire analysis of Data Independent Acquisition. Note that Mascot requires a commercially available license. Select at least one search engine to be used. If more than one search engine is selected, ProHits will automatically use the iProphet scheme to combine all results into a single ProteinProphet output.

Database: The databases are set in the Admin module of ProHits in FASTA format. They are given unique identifiers. From the list of databases available in ProHits, an administrator can control which ones are visible to the users by selecting the page edit () icon from list of the databases (accessible through the “View” button on the New Task page).




Modifications: Available modifications are shown on the right pane; select desired modifications (fixed or variable) using the arrows. An administrator can select which modifications are available to all users or for selected users (first select the "Set default modifications" page edit () icon and use arrows to add desired modifications).


Besides the Search Engine Parameters described above, this New Search page enables you to set your raw file conversion parameters and to trigger analysis of the results through the Trans Proteomics Pipeline (TPP) at the Institute for Systems Biology; PeptideProphet, ProteinProphet and iProphet analysis will be performed.

Automatic conversion of vendor file formats to mgf, mzML and mzXML files will be performed (through vendor specific conversion software and ProteoWizard); default conversion parameters can be overridden. You also have the option of automatically running the TPP statistical software tools on the search results. If you choose to do so, select the “Run TPP” box and parameter set. Note that you can also run the TPP tools after search results are returned.

Once the parameters are selected, add the files to be analysed by clicking “Add files”. This will open the folders for the specified instrument, allowing you to select as many files as desired for searching. Each user can only have one running task per instrument; additional tasks (unlimited number) will be placed in a queue. The searches will automatically initiate (in the order they were queued) once the initial searches are completed.

Once all parameters are selected, click the “Run Task” button at the bottom of the screen. The searches will be initiated. Progress can be monitored from the instrument Auto Search home page through “Running Task Status” and the Search and TPP logs. Searches in progress will also be highlighted in the Auto Search home page, as shown below. Tip: instead of creating a new search task for each file you analyze using the same search parameters, use the “modify” option at the bottom of the page of an existing search task: this will group all your searches within the same search task folder, making it easier to retrieve, and will also ensure that the same search parameters are used for each file.

Search Task view

=> “Instrument Search Task”

Searches that are currently being searched will be highlighted in green while search tasks that are in the queue are highlighted in blue. Yellow highlights an error with the search. No other searches can be initiated until errors are resolved, either through successful running of the problematic search, or by stopping the task. IMPORTANT: If an error is encountered while tasks are in the queue, one of these tasks must be manually re-started (the other Tasks will then be searched in queue).


=> Select the page modification icon in the Task Detail column.

This will provide an overview of the search parameters used (click on [+] as shown here for Comet toexpand the parameter list). These can be used for reporting.

While the parameters cannot be changed once a task has been initiated, the task itself can be modified through the addition of new files to be searched. Selecting “Modify Task” at the bottom will open a new window, identical to the “New Task” window except that the only available option is the addition of new files.



View Search Results

=> Select the “Result Detail” for one search task to view search results

This opens up a new page. Tracking of the file folders, search parameters, etc., is provided on the top half of the page.



The search results are displayed at the bottom of the page. The blue links connect to the search engine or TPP pages (original format). You can also select to analyse results using TPP (if not performed initially by clicking [New], or if the analysis had failed by simply select the repeat ( icon) and identify files to be analysed.


Merging samples prior to TPP analysis

There are cases where youmay want to merge files prior to running the TPP (examples include fractionated samples, gel-based or otherwise). To do this in ProHits the searches must be performed using the same search engine (i.e. Mascot and X!Tandem files cannot be combined) and the results must be located within the same Search Task folder. Select the “Merge” optionand the files to be combined.


Selecting the “Merge” option will create a new entry at the bottom of the page. The TPP can be run on this entry in the same fashion as on individual files, by selecting the desired parameters and pressing “Run TPP” At this point, we have performed database searching and have obtained results directly from the search engines, as well as results from the TPP. We are now ready to transfer these search results into the “High-Throughput Analyst” module.

The transfer involves two steps: 1) linking the initial file to an entry created in the Analyst module; and 2) parsing the search results.

Linking files to Analyst

=> link the file

Autolink: The easiest way to link a raw mass spectrometry file (and all associated converted files and search results) to a sample defined in the “Analyst” module is to first create “Samples” in the Analyst and to follow a simple naming convention for both raw files and folders on the acquisition computer. While this will be discussed in more detail in the Analyst module, the naming convention of the folders is ANY_FOLDER_NAME_Pxx where Pxx is a project number defined in the Analyst module (e.g. P17 in the example below). The raw file naming convention is to begin with the unique identifier assigned to a Sample in the Analyst module, as well as the four first characters of the user-entered Sample name (e.g. the four first characters of a gene name); underscore separates these entries. In this example, “2376” is the unique identifier and “TOP3” are the characters recognized. Providing that the file 2376_TOP3… is in the proper folder on the acquisition computer, a green “Autolink” icon () will be shown:



You may have already linked the native file (not searched) to the Analyst modules through the “Storage” area. If so, the “link” icon by the file size will be shown in yellow. A white link indicates that no link has been established at this point.


To manually link the raw file to an entry in the Analyst module, click on a white icon: this triggers a pop-up window where a pre-entered bait, experiment and sample set can be selected. New baits, experiments and samples can also be created by clicking on the [new] links within this pop-up window. How to create such entries will be detailed in the Analyst module.

Parsing files to the Analyst module

You are now ready for parsing (transferring results to the Analyst module), as you have a linked file (yellow icon) and search results. In this case you also have TPP results. Both types of results can be parsed.

=> Select the “Detail” link in the parsing area

You can transfer (parse) results from the search engines, from the TPP or both. You can select the parameters for the parsing cut-offs for protein inference. For the gene level parser, the cutoffs are applied at the peptide level. Press “Run to initiate parsing. A “processing” status notification will appear.


When parsing is successfully completed, a green check mark will appear in place of the checkbox.

In the event that you decide to link or parse different file(s) to the same Analyst entry, you can remove the parsed files or the link: alongside the tick box, a square containing a red X can be selected if you want to remove the hits from the Analystmodule. Note that you can also unlink a sample by clicking on the yellow link icon and selecting “remove link”.


Note that if you press the [Detail] button located inside the parsing box, you will see the details of the parsing options.


4. Auto Search for Data Independent Acquisition

ProHits supports identification from Data Independent Acquisition files through both DIA-Umpire and MSPLIT-DIA.

Running DIA-Umpire untargeted identification through the ProHits interface:

First select “DIA file” and “MS-Umpire”. In this Data Management module of ProHits, DIA-Umpire SE (Signal Extraction) generates pseudo-MS/MS spectra from precursor-fragment group data. These spectra can be searched using standard tools designed for DDA analysis, including the iProphet pipeline (see previous section for details); note that DIA-Umpire SE extracts three quality tiers of precursor-fragment group data that are each searched and modeled separately (see schema below). The final iProphet ProteinProphet result merges all quality groups for all search engines used. Currently, Comet, MSGF+, X!Tandem and Mascot are supported. Select the desired database search engines, protein database and search parameters that will be used to search the pseudo MS/MS spectra. Select the DIA-Umpire parameters. DIA-Umpire supports a variable window design (see Tsou et al., Nature Methods, 2015). In this case, specify each individual window’s boundaries. Leave blank if fixed windows are used.


The following figures present the detailed process for combining search results in the DIA-Umpire SE module. For the iProphet pipeline: DIA-Umpire separates the precursor-fragment groups in three quality tiers. Q1 corresponds to MS1 clusters of 3 or more isotope peaks, Q2, to those with 2, and Q3 to those that have been detected only as clusters of at least 2 unfragmented precursor ion peaks in MS2. Because of the different ratios of correct versus incorrect identifications across these tiers, PeptideProphet is run separately for each of the tiers (see details in DIA-Umpire original paper by Tsou et al., Nature Methods 2015). To combine results from multiple search engines into one final report, searches using each search engine are first performed on individual quality tiers, and PeptideProphet is performed on those results. InterProphet parser is used to combine the results into a single pep.xml file for each of the quality tiers, and once again to combine across all search engines and all quality tiers (this is the version used for extraction of gene level information). iProphet (ProteinProphet) is then applied to these three combined pep.xml files to yield the final protein list. Note that the gene level viewer instead extracts information directly from the pep.xml to perform mapping at the gene level (grey).


In addition to providing an iProphet view of the data, pep.xml and prot.xml files are generated for each search engine. In this case, PeptideProphet is run on the individual files, which are combined into a single pep.xml using the InterProphet parser (gene level information is extracted from the resulting file). ProteinProphet runs directly on the Q1, Q2 and Q3 pep.xml outputs to generate the prot.xml file.


Running MSPLIT-DIA identification through the ProHits interface:

Select “DIA file” and “MSPLIT-DIA”. If using DDA files to create a library, select DDA search parameters (only MSGFDB is currently supported as a search engine). Add both DDA and DIA (SWATH) files to the list of files and select which of those are the DDA files that will be used for library generation. Existing libraries can also be searched: individual external libraries are listed (here Human_Swath_Atlas_v1 from the Aebersold group). In addition, ProHits automatically generates a spectral library from all previously-searched DDA files that it appends to externally generated libraries (“Archive”). These can be searched in isolation, or alongside the newly searched results. In all cases, ProHits will take the highest MSGFDB spectrum for spectral matching with MSPLIT-DIA. Select the parameters to be used for creating the spectral library from MSGFDB results, creatingthe decoy spectral library, performing the spectral library search and filtering the search results using retention time correlation. MSPLIT-DIA supports a variable window design (although this was not specifically described in Wang et al., Nature Methods, 2015). In this case, specify each individual window’s boundaries. Leave blank if fixed windows are used.

The MSPLIT-DIA search results page differs somewhat from that of the DDA and DIA-Umpire results. Currently, the TPP is not yet enabled for MSPLIT-DIA and gene level inference mapping is performed through a simple set of rules. The identification results for each file can be downloaded as a tab delimited file (download “ ” icon by each file name). ProHits also facilitates the generation through MSPLIT-DIA of “assay libraries” for targeted extraction by the popular tools OpenSWATH, PeakView and Skyline.
